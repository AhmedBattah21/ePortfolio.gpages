Enhancement Three



  • Enhancement Three: Database

    A database is a collection of information organized in a specific format. In my previous code review, I had mentioned how the use of file based database has weaken the application. This application uses a file based database, SQLite database. According to the review, it is difficult to achieve data sharing and also app data is insecure.

    Therefore I enhanced this application by considering usage of a local server based database. The database system will be installed in one of the machines in the local network or can be hosted in a remote computer. The application will therefore connect to the database to retrieve and write data during run time.

    Also, the loading of data directly into user interfaces caused the application to load slowly at start up. To remove the weakness, I have added a code which loads the data once the application has started. This part of the code will retrieve only ten records at start up, whenever user wish to see more, he/she will click a button to load more data. This will make the app load faster at start up.

    The enhancements listed in my previous code review have been met and the program is now improved. Searching of data is quick and easy. The application does not freeze at adminâ€™s log in. However I am still checking the best algorithm to use so that all users, regardless of the memory capacity of their computers, should have the application working and meeting its objectives.
